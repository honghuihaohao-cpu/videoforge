## 三、架构设计 — 80/100

### 亮点

1. **工作流引擎独立于 UI**：`WORKFLOW_STEPS` 数组派生出所有工作流页面，添加新步骤只需在数组中追加一条记录，无需改其他代码。

2. **API 设计遵循 RESTful**：
   - `GET/POST /api/projects` — 列表和创建
   - `DELETE /api/projects?id=xxx` — 删除
   - `POST /api/projects/steps` — 步进进度保存
   - `GET /api/platform/bilibili` — 外部数据代理

3. **数据模型设计合理**：`Project → WorkflowStep[] / Analysis[] / VideoData / ImprovementLog[]` 一对多关系清晰，级联删除避免孤儿数据。

4. **平台 API 封装**：`lib/platform/bilibili.ts` 独立封装，换了平台不影响核心逻辑。

5. **SEO/SSR 兼顾**：首页、项目页用 Server Component（数据直查），交互页（工作流步骤、数据分析）用 Client Component，水合边界合理。

### 风险点

- 无 API 限流，单用户脚本可能打满数据库连接
- SQLite 并发写入瓶颈（单文件锁），不适合 >50 个并发用户
- 无请求日志/监控/告警
- 没有数据库迁移回滚策略

## 四、安全性 — 75/100

### 已做对的事

- API Key 不硬编码，通过 env 传入
- Zod 校验防止 SQL 注入（Prisma 参数化查询天然防注入）
- 敏感操作（删除项目）有确认弹窗
- `.env` 被 .gitignore 排除，不会意外提交
- 生产构建的 `process.env.NODE_ENV` 判断到位

### 需要警惕

- **CSRF 保护缺失**：API routes 无 CSRF token 校验
- **无认证机制**：任何人部署后所有 API 公开可调
- `apiKey` 参数可以从客户端传入（虽然是为了测试便利，但暴露了 key 传输路径）
- **B 站 API 调用无缓存**：可能触发 B 站反爬
- **错误信息中可能泄露内部路径**：`details: parsed.error.flatten()` 在生产中暴露了 schema 结构

## 五、可部署性 — 85/100

### 便利之处

- SQLite 零配置数据库，无需安装 Postgres/MySQL
- `npm install → npx prisma migrate dev → npm run dev` 三步启动
- Vercel 可一键部署（Next.js 原生支持）
- Docker 可容器化（未来加 docker-compose.yml 即可）
- `.env.example` 清晰标注所需环境变量
- MIT 协议，无法律风险

### 部署注意事项

- Vercel 部署时 SQLite 文件会丢失（无状态），需改用 Turso 或 Vercel Postgres
- 生产环境需关闭 Zod 的 `details` 泄露
- 需设置 `NODE_ENV=production`

## 七、市场与竞品分析

### 目标用户

知识科普视频创作者（B 站 UP 主 / 抖音创作者 / 视频号博主），单人 or 小团队，月更 2-8 条深度内容。

### 竞品格局

| 竞品 | 类型 | VideoForge 对比 |
|------|------|----------------|
| Notion 模板 | 纯文档 | VF 有 AI 评估 + 数据集成 |
| 飞书多维表格 | 协作表格 | VF 有结构化工作流 + AI |
| 剪映/Pr 剪辑 | 剪辑工具 | VF 专注前期策划和后期复盘 |
| 新榜/火烧云 | 数据平台 | VF 轻量 + 开源 + 可自部署 |
| ChatGPT 直接问 | AI 对话 | VF 提供系统化工作流而非零散问答 |

**定位**：VideoForge 不替代任何单一工具，它填补的是"从策划到复盘"的**流程管理**空白——竞品要么只做数据，要么只做剪辑，没人在做"全流程 + AI 伴随"。

### 核心风险

- **依赖 Claude API**：如果用户没有 API Key，AI 分析功能完全不可用。需要支持更多模型（OpenAI / 国产模型）。
- **平台 API 壁垒**：抖音和视频号没有公开 API，数据拉取依赖手动录入，体验割裂。
- **单用户设计**：无多用户/团队协作，限制了传播（用户无法推荐给团队一起用）。
